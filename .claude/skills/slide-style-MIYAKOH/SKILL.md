# slide-style-MIYAKOH

既存の Marp スライドに MIYAKOH スタイルのレイアウトパターンを適用して整形する。

スライドの内容作成は行わない。ユーザーが書いた Marp スライドを受け取り、`slides/example.md` の39種パターンから最適なものを選んで適用することで、デザインの一貫性と品質を確保する。

## 基本原則

- `slides/example.md` の39種パターンから選んで適用する。独自レイアウトは作らない
- 見出しはアクションタイトルに書き換える（「コスト分析」→「ツールAは3年TCOで30%有利」）
- コロン（：）、感嘆符、装飾的な絵文字を使わない
- アクセントカラーは1スライド1-2色まで
- `stat-box accent`（Teal背景）は並列要素の中で1つだけ強調したい場合のみ。全要素同列なら全部 `stat-box`（グレー）で統一

## 参照ファイル

| ファイル | 内容 | 参照タイミング |
|---------|------|-------------|
| `slides/example.md` | 39種パターンの実装コード | **整形時（必須）** |
| `docs/pattern-reference.md` | パターンの役割・選択ガイド | パターン選択時 |

## ワークフロー

### Step 1: 対象スライドを読む

ユーザーが指定した Marp ファイル（と対象スライド範囲）を読み込み、各スライドの内容と現在の構造を把握する。

### Step 2: パターンを選ぶ

`docs/pattern-reference.md` の選択ガイドを参照し、各スライドの内容に最適なパターンを決める。

選択の判断基準:
- そのスライドは何を伝えようとしているか
- 要素の数（2つの比較 → Pattern 6、3つの並列 → Pattern 8）
- 情報の種類（数値 → Pattern 27、手順 → Pattern 14/15）

### Step 3: パターンを適用する

`slides/example.md` から該当パターンの Markdown をコピーし、スライドの内容で書き換える。

適用ルール:
- `<!-- _class: xxx -->` でクラス指定
- div レイアウトはユーティリティクラス使用（`grid grid-cols-2 gap-6` 等）
- 同じパターンを3回以上連続で使わない
- 出典は `<div class="source">出典: ...</div>`
- `themes/MIYAKOH.css` がデッキと同じディレクトリにあることを確認し、なければコピーする

## 画像生成（slimg 連携）

パターン適用時に画像スロットがある場合（Pattern 7a/7b, 8, 12, 18, 22, 24, 26 等）、`/slimg` で MIYAKOH スタイルに合った画像を生成する。

```bash
npx @miyakoh/slimg "<プロンプト>" -t flat -a <ratio> -o images/<name>.png
```

スタイル統一ルール:
- プロンプトに `flat design, navy #1B4565 and teal #3E9BA4, white background` を含める
- 保存先: デッキと同じディレクトリの `images/`

パターン別アスペクト比:

| パターン | 比率 | 用途 |
|---------|------|------|
| 7a/7b, 8, 22 | 4:3 | カラム内画像 |
| 18 | 16:9 | パネルヘッダー |
| 24, 26 | 3:4 | 右配置・分割背景 |
| 12 | 1:1 | グリッド内小画像 |

## 判断に迷ったら

- スライドを分ける
- Pattern 6 (2カラム比較) か Pattern 8 (3カラム) を選ぶ
- 表・グラフは Pattern 39 (テーブル) か Pattern 27 (統計)
- 装飾より明快さを優先
