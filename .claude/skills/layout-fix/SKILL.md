# layout-fix

Marp スライドのレイアウト崩れを**視覚的に**検出し、修正する。Playwright でヘッドレス Chromium を起動し、実際のブラウザレンダリング結果をスクリーンショットで確認することで、コードだけでは見えない表示の問題を発見する。

---

## ワークフロー

### Step 1: スクリーンショット生成

`scripts/check_layout.py` を使い、Marp Markdown を HTML にビルドしてから Playwright でスライドごとにスクリーンショットを撮る:

```bash
uv run --with playwright python skills/layout-fix/scripts/check_layout.py <deck.md のパス>
```

- テーマ CSS（`MIYAKOH.css`）はデッキと同じディレクトリから自動検出される
- スクリーンショットは一時ディレクトリ（`/tmp/marp-layout-*`）に `slide-001.png`, `slide-002.png`, ... として保存される
- 出力の `SCREENSHOT_DIR=...` 行でディレクトリパスが分かる

**オプション:**

| オプション | 説明 |
|-----------|------|
| `--theme <path>` | テーマ CSS を明示指定（自動検出できない場合） |
| `--cleanup <dir>` | 指定ディレクトリを削除して終了（後片付け用） |

**初回セットアップ:** Playwright のブラウザがインストールされていない場合は先に実行:
```bash
uv run --with playwright python -m playwright install chromium
```

### Step 2: 視覚チェック

各スクリーンショットを Read ツールで1枚ずつ確認し、以下の問題を検出する:

| 問題カテゴリ | チェック内容 |
|------------|------------|
| **はみ出し** | テキスト・要素がスライド境界を超えている |
| **余白不足** | 端に寄りすぎ、コンテンツが詰まりすぎ |
| **カラム崩れ** | グリッドが崩壊し、要素が重なっている |
| **フォントサイズ** | 文字が小さすぎて読めない（プロジェクター表示を想定） |
| **空白過多** | コンテンツに対してスライドが空きすぎ |
| **配置の不整合** | 要素の垂直・水平位置が揃っていない |
| **色のコントラスト** | 背景色と文字色が近すぎて読みにくい |

各スライドについて、問題がなければ「OK」、問題があれば具体的な箇所と症状を記録する。

### Step 3: 修正方針の決定

検出した問題ごとに修正方針を選ぶ:

| 問題 | 修正方針（優先順） |
|------|-----------------|
| テキストはみ出し | ① テキスト量を減らす ② `compact` クラスを追加 ③ スライドを2枚に分割 |
| リスト項目が多い（5項目以上） | ① 項目を削減 ② 2カラムグリッドに変換 |
| 余白不足 | padding/margin ユーティリティクラスを追加（`p-4`, `mt-4` 等） |
| カラム崩れ | `grid-cols-N` や `gap-N` の値を調整 |
| 空白過多 | コンテンツを追加 or 別パターンに変更 |
| 配置の不整合 | `items-center`, `justify-center` 等のフレックスクラスを追加 |

### Step 4: 修正の適用

**修正するもの:** デッキの Markdown ファイルのみ。スキルファイルやテーマ CSS は変更しない。

**修正の原則:**
- 1スライドずつ修正する。一括で変更しない
- 構造（パターン選択）の変更よりも、クラス調整を優先する
- テキスト量の削減が最も効果的な修正手段であることが多い
- 迷ったらスライドを分割する（1スライド1メッセージ）

### Step 5: 再確認

修正後、再度スクリーンショットを生成して確認する:

```bash
uv run --with playwright python skills/layout-fix/scripts/check_layout.py <deck.md のパス>
```

問題が解消されるまで Step 2〜5 を繰り返す。通常1-2回の修正サイクルで完了する。

### Step 6: 後片付け

確認が完了したら一時ディレクトリを削除する:

```bash
uv run --with playwright python skills/layout-fix/scripts/check_layout.py --cleanup <SCREENSHOT_DIR>
```

ビルドで生成された HTML ファイル（`deck.html`）も不要なら削除する。

## よくある修正パターン

### A. テキストはみ出し → compact クラスの追加

`compact` クラスはフォントサイズを全体的に縮小し、情報密度の高いスライドに対応する。

```markdown
<!-- 修正前 -->
# 長いタイトルと多くの本文

<!-- 修正後: _class に compact を追加 -->
<!-- _class: compact -->
# 長いタイトルと多くの本文
```

### B. 長いリスト → 2カラム化

5項目以上のリストが縦に収まらない場合、グリッドで2列に分割する。

```markdown
<!-- 修正前 -->
- 項目1
- 項目2
- 項目3
- 項目4
- 項目5
- 項目6

<!-- 修正後 -->
<div class="grid grid-cols-2 gap-4">
<div>

- 項目1
- 項目2
- 項目3

</div>
<div>

- 項目4
- 項目5
- 項目6

</div>
</div>
```

### C. パネル内のはみ出し → gap とパディングの調整

パネル内のコンテンツが溢れる場合、gap を縮小しパディングを調整する。

```markdown
<!-- 修正前 -->
<div class="grid grid-cols-3 gap-8">

<!-- 修正後: gap を縮小 -->
<div class="grid grid-cols-3 gap-4">
```

### D. テーブルのはみ出し → compact クラスまたは列の削減

```markdown
<!-- 修正前: 列が多すぎてはみ出す -->
| A | B | C | D | E | F |

<!-- 修正後①: compact クラスでフォント縮小 -->
<!-- _class: compact -->
| A | B | C | D | E | F |

<!-- 修正後②: 列を減らす -->
| A | B | C | D |
```

### E. 画像と文字の重なり → 画像サイズ調整

```markdown
<!-- 修正前: 画像が大きすぎて文字と重なる -->
![bg right:50%](image.png)

<!-- 修正後: 画像領域を縮小 -->
![bg right:35%](image.png)
```

### F. スライド分割

1枚に詰め込みすぎている場合、最も効果的な修正は分割。

```markdown
<!-- 修正前: 1枚に2つの論点 -->
# 課題と対策
## 課題
...
## 対策
...

<!-- 修正後: 2枚に分割 -->
# 3つの課題が成長を阻んでいる
...

---

# 課題に対する3つの対策
...
```

## チェックリスト

修正完了後、最終確認として以下を確認:

- [ ] 全スライドでテキストがスライド境界内に収まっている
- [ ] 余白が適切（上下左右に最低 40px 程度の余白）
- [ ] グリッド・カラムが意図通りに表示されている
- [ ] フォントサイズがプロジェクター表示で読める大きさ（最低 18px 相当）
- [ ] 表紙・セクション扉・クロージングのグラデーション背景が正しく表示
- [ ] 画像が正しく読み込まれている（壊れた画像アイコンがない）
